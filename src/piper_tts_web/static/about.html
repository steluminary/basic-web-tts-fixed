About Basic TTS

Basic TTS is a fast, free, and easy-to-use text-to-speech tool originally created by Patrick Metzger. It was built as a simple web interface on top of Piper so you can convert text to natural-sounding speech using high-quality English voices.

The original hosted version eventually stopped working reliably because of Firebase-related changes. This fork exists to keep the idea alive as a self-hosted, Docker-based Piper host and web GUI that you can run locally or on your own server. The goal is a small, focused tool for CPU-based text-to-speech, without external cloud dependencies.

On the web UI, you type text, pick a voice, and generate speech. You can listen in the browser or download the resulting audio. It is useful for accessibility, content creation, testing, and other local TTS workflows.

The original source code and project description can be found in Patrick Metzger’s repository. This fork, maintained at github.com/steluminary/basic-web-tts-fixed , focuses on keeping the app working with Piper and making it straightforward to self-host.

Key Features

  • Multiple voices: Choose from a range of English voices, each with its own accent and style.
  • Instant conversion: Enter your text, select a voice, and generate speech audio in seconds.
  • File management: Each conversion produces a WAV file and an MP3 version automatically, so you can download either format for use in editors, games, or other tools.
  • Audio controls: Play and pause in the browser and scrub through the generated audio; then download the file when you are satisfied with the result.
  • Progress feedback: See progress as your text is processed and converted.
  • Open source and self-hosted: The project is open source and designed to be easy to deploy, modify, or extend on your own hardware.

Sample Voices and Benchmarks

To help you choose a voice, there is a separate benchmark page with sample audio and timing metrics for many Piper voices. You can listen to each sample and see generation time, realtime factor, and file sizes side-by-side.

Installing Voices from Hugging Face

Basic TTS uses Piper voice models stored in the models directory inside the container (mapped to a host directory via Docker). You can download additional voices from the Piper voices collection on Hugging Face: https://huggingface.co/rhasspy/piper-voices .

For each voice you want to add, download the .onnx model file and its matching .onnx.json config file into your host-side models directory. When the container starts, Basic TTS scans that directory and makes the installed voices available in the voice dropdown, so you can try them immediately without rebuilding the image.

How It Works (Web UI)

  1. Select a voice from the available voices in the dropdown menu.
  2. Type or paste the text you want to convert.
  3. Click “Convert to Speech” and wait for the audio to be generated.
  4. Use the built-in audio player to listen and, when ready, download the WAV or MP3 file.

Note: The built-in audio player provides basic playback controls (play/pause and scrub), but does not currently include a playback speed slider.

Self‑Hosting and Docker Layout

Basic TTS is intended to be self-hosted. A typical Docker setup uses a single container that runs the Piper backend and the web interface together. The core directories inside the container are:

  • /app/src/piper_tts_web/models – where Piper voice model files are stored.
  • /app/src/piper_tts_web/outputs – where generated audio files (WAV/MP3) are saved.
  • /app/start.sh – startup script used to configure and launch the app.

These paths are usually mapped to host directories so you can manage models, keep outputs, and tweak startup behavior without rebuilding the image.

Building the Docker Image

The repository includes a Dockerfile that builds an image containing the Piper backend and the Basic TTS web interface. To build the image manually, clone the forked repository and run: 

git clone https://github.com/steluminary/basic-web-tts-fixed.git
cd basic-web-tts-fixed
docker build -t piper-tts-web:latest .

A pre-built image on Docker Hub may be provided in the future for an even smoother docker run or Docker stack experience. If that becomes available, the GitHub repository will include updated instructions and the Docker Hub image name so you can pull and run without building locally.

Example docker-compose.yml

Once you have a local image (or a future Docker Hub image), you can orchestrate the container with a docker-compose.yml similar to this: 

version: "3.8"
services:
  basictts:
    image: piper-tts-web:latest
    container_name: basictts
    restart: unless-stopped
    ports:
      - 8888:8000
    volumes:
      - /path/to/basictts/models:/app/src/piper_tts_web/models
      - /path/to/basictts/outputs:/app/src/piper_tts_web/outputs
      - /path/to/basictts/config/start.sh:/app/start.sh:ro
    mem_limit: 3g

Adjust /path/to/basictts/models, /path/to/basictts/outputs, and /path/to/basictts/config/start.sh to match your own filesystem layout. The idea is to keep models, outputs, and configuration on the host so you can update voices and collect audio without rebuilding the image. The start.sh file is mapped in as read-only so you can tune runtime parameters (for example, the number of workers or other Piper options) without rebuilding the Docker image itself.
