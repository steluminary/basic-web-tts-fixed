About Basic TTS

Basic TTS is a fast, free, and easy-to-use text-to-speech tool originally created by Patrick Metzger. It was built as a simple web interface on top of Piper so you can convert text to natural-sounding speech using high-quality English voices.

The original hosted version eventually stopped working reliably because of Firebase-related changes. This fork exists to keep the idea alive as a self-hosted, Docker-based Piper host and web GUI that you can run locally or on your own server. The goal is a small, focused tool for CPU-based text-to-speech, without external cloud dependencies.

On the web UI, you type text, pick a voice, and generate speech. You can listen in the browser or download the resulting audio. It is useful for accessibility, content creation, testing, and other local TTS workflows.

The original source code and project description can be found in Patrick Metzger’s repository. This fork, maintained at github.com/steluminary/basic-web-tts-fixed , focuses on keeping the app working with Piper and making it straightforward to self-host.

Key Features

  • Multiple voices: Choose from a range of English voices, each with its own accent and style.
  • Instant conversion: Enter your text, select a voice, and generate speech audio in seconds.
  • Audio controls: Play, pause, and download the generated audio for offline use.
  • Progress feedback: See progress as your text is processed and converted.
  • Open source and self-hosted: The project is open source and designed to be easy to deploy, modify, or extend on your own hardware.

How It Works (Web UI)

  1. Select a voice from the available voices in the dropdown menu.
  2. Type or paste the text you want to convert.
  3. Click “Convert to Speech” and wait for the audio to be generated.
  4. Use the built-in audio player to listen or download the result.

Note: The built-in audio player provides basic playback controls (play/pause and scrub), but does not currently include a playback speed slider.

Self‑Hosting and Docker Layout

Basic TTS is intended to be self-hosted. A typical Docker setup uses a single container that runs the Piper backend and the web interface together. The core directories inside the container are:

  • /app/src/piper_tts_web/models – where Piper voice model files are stored.
  • /app/src/piper_tts_web/outputs – where generated audio files (WAV/MP3) are saved.
  • /app/start.sh – startup script used to configure and launch the app.

In a typical docker-compose.yml, these are mapped to host directories like: 

version: "3.8"
services:
  basictts:
    build:
      context: /mnt/home/docker/basictts/src
    image: piper-tts-web:latest
    container_name: basictts
    restart: unless-stopped
    ports:
      - 8851:8000
    volumes:
      - /mnt/home/docker/basictts/models:/app/src/piper_tts_web/models
      - /mnt/home/docker/basictts/outputs:/app/src/piper_tts_web/outputs
      - /mnt/home/docker/basictts/config/start.sh:/app/start.sh:ro
    mem_limit: 3g

You can adjust these paths to match your own environment, but the idea is to keep models, outputs, and configuration on the host so you can update voices and collect audio without rebuilding the image. The start.sh file is mapped in as read-only so you can tune runtime parameters (for example, the number of workers or other Piper options) without rebuilding the Docker image itself.

Building and Running the Docker Image

The repository includes a Dockerfile that builds an image containing the Piper backend and the Basic TTS web interface. To build the image manually, clone the forked repository and run: 

git clone https://github.com/steluminary/basic-web-tts-fixed.git
cd basic-web-tts-fixed
docker build -t piper-tts-web:latest .

After the image is built, you can use a docker-compose.yml similar to the one shown above to start the container, map your model and output directories, and expose the web interface on a port like 8851.

A pre-built image on Docker Hub may be provided in the future for an even smoother “docker stack” or docker run experience. If that becomes available, the GitHub repository will include updated instructions and the Docker Hub image name so you can pull and run without building locally.

Technology

  • Backend: Python, using Piper for TTS synthesis.
  • Frontend: JavaScript, HTML, and CSS for a simple, responsive web interface.
